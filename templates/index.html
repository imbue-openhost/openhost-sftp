Connection details

The SSH/SFTP service listens at:

{{ ssh_user }}@{{ ssh_host }}, port {{ ssh_port }}

Examples:

sftp -P {{ ssh_port }} {{ ssh_user }}@{{ ssh_host }}
rsync -e "ssh -p {{ ssh_port }}" -avz local/ {{ ssh_user }}@{{ ssh_host }}:files/
ssh -p {{ ssh_port }} {{ ssh_user }}@{{ ssh_host }} 
    

      The third form (TTY-less ssh) is rejected by design —
      PermitTTY no + ForceCommand internal-sftp
      mean only the SFTP subsystem is available.  Files written via SFTP
      land under the files/ subdirectory inside your chroot.
